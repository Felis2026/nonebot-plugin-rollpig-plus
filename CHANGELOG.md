# 更新日志

## v0.9.0

### 图片版图鉴渲染

- 图片版小猪图鉴改为纯 Pillow 绘制，保留分页、EX Lv.、NEW / MAX 标记、统计信息和结果缓存。
- 重做半透明毛玻璃卡片、立体等级胶囊和顶部统计布局，并优化 Windows、Linux、Docker / WSL2 下的中文字体加载。
- 图鉴绘制移入工作线程并保留有界并发，避免阻塞 NoneBot 事件循环或在突发请求下产生无上限的内存峰值。

### 依赖与兼容

- 移除图鉴 HTML 模板、Playwright 页面池、截图后处理及 `nonebot-plugin-htmlrender` 依赖。
- 保持图鉴命令、业务快照、用户数据和资源目录兼容，现有部署升级后不需要迁移数据。

## v0.8.2

### 多私有资源包

- 新增 `rollpig_private_resource_manifests`，支持按顺序叠加多个私有 overlay 资源包。
- 旧版 `rollpig_private_resource_manifest_url` / `rollpig_private_resource_token` 继续兼容，会作为 legacy overlay 追加加载。
- 私有包支持填写本地 `manifest.json` 路径或 `file://` URL，方便用户在本机维护自己的小猪资源包。
- 多私有包使用独立缓存目录，单个私有包同步失败时不会影响公有资源或其它私有包。

### 资源包扩展

- 可同时启用公开资源包、官方 GIF 包、PJsk 私有包和用户自建私有包。
- 官方 GIF 动态小猪 overlay 不走用户配置，会随云端资源同步固定叠加。
- 私有 overlay 继续要求新增猪与已有 ID 不冲突；覆盖已有猪需要使用 `pig_overrides.json` 明确声明。

## v0.8.1

### 每日总结与分群控制

- 每日总结默认改为关闭，避免新部署实例在管理员未确认前主动向群聊推送定时消息。
- 新增 `小猪日报` / `小猪日报 状态` / `小猪日报 开启` / `小猪日报 关闭` 命令，支持群主或管理员控制本群，SUPERUSER 可追加群号控制其他群。
- `rollpig_daily_summary_enabled` 调整为未被单群命令或外部控制器覆盖时的默认值；需要全局默认开启时可显式设为 `true`。
- 新增日报控制器接口；公开版无外部控制器时写入插件本地数据，宿主项目接入外部控制器时可保持单一状态源。
- 日报定时任务先筛选已开启日报的群，再生成日报与次日保护名单，避免关闭日报的群被定时任务产生额外副作用。

## v0.8.0

### 普通卡片渲染

- `今日小猪`、`昨日小猪`、`今日烤猪`、`烤群友` 等普通卡片改为 Pillow 渲染，降低普通图片输出对 Chromium / Playwright 的依赖。
- 内置 Source Han Sans SC Medium 作为默认 CJK 字体，并保留 `rollpig_card_font_path` 供用户自行指定字体。
- 内置 Google Noto Emoji 32px ZIP，普通卡片可离线渲染彩色 Emoji，减少 Docker / Linux 环境缺字体导致的方块字或黑白 Emoji。
- 普通卡片保持 800×800 输出，头像资源统一规整到 240×240 区域；图片版小猪图鉴仍沿用 HTML/CSS 渲染。

### GIF 动态资源

- 普通卡片支持 GIF 小猪资源，`今日小猪`、`烤猪`、`烤群友` 等图片输出会随资源自动生成 GIF。
- 资源图片支持 `.png` / `.gif`，同 ID 同时存在时优先使用 GIF。
- 图片版小猪图鉴固定取 GIF 首帧缩略图，保持静态陈列。
- GIF 输出使用全帧采样调色板，兼顾文字稳定和动态头像色彩保留。

### 内置资源更新

- 将内置公共资源同步到 `rollpig-resources/rollpig` 的 `2026-06-28.1` 版本，默认小猪数量更新至 165 只。
- 新增 `猪水群`、`猪诞树`、`猪龙鱼公爵`、`肥猪`、`猪健身`、`狮子猪`、`猪泥好运`、`护士小猪`、`油画猪`、`猪兔同笼`、`猪打滚`、`震撼美味`、`小心猪滑`、`抽搐猪`、`飞碟猪`、`猪意味`。
- 同步最新 `pig_rules.json`，保持熟食、特殊形态、烤猪排除等规则与公共资源包一致。

### AI 烤猪

- 默认模型切换为 `deepseek-v4-flash` 非思考模式。
- 自动兼容旧 `deepseek-chat` / `deepseek-reasoner` 配置，避免 DeepSeek V4 迁移期旧配置直接失效。

## v0.7.4

### 稳定性修复

- PigHub 图库刷新增加 single-flight、失败冷却与旧缓存回退，避免并发缓存击穿。
- `roast_library.json` 写入改为异步锁、线程落盘与临时文件原子替换，降低事件循环阻塞和 JSON 损坏风险。
- 启动期资源同步后台任务纳入 shutdown 生命周期，避免关闭期仍在改动资源缓存目录。

### 并发与数据边界

- 图鉴缓存增加同 key single-flight，避免多个相同请求重复占用 Chromium 渲染资源。
- 本地数据读取接口返回副本，避免命令层误改内部 list / dict 后绕过锁与原子保存。
- 旧烤群友 `check_roast_usage()` / `update_roast_usage()` 标记为 deprecated，后续新流程统一使用原子扣减接口。
